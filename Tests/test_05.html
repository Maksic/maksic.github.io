Negative testing
ЛР05
Студентам надо будет написать проверки по негативному тестированию, провести его (в Google Chrome) и занести баги. В итоге за лабораторную работу студент получит две оценки:
1. За проверки
2. За баги
Сайт для тестирования: Belavia.by
Форма, которую нужно протестировать: Book Flights
Баги: оформляются в той же системе (форме), как и в лабораторной работе №2 «Баг репорты».
Assignee = преподаватель занятия по Негативному тестированию.
Если баг не относится к негативному тестированию, он не оценивается.
Нужно написать только негативные проверки этой формы. Это проверки, показывающие, что система корректно обрабатывает недопустимые действия пользователя. Обратите внимание, что проверки в этом тестировании становятся сложнее. Они не привязаны только к одному полю. Поля на форме Book Flights взаимозависимые. Поэтому чтобы провести негативное тестирование, не достаточно одного действия. Нужно выполнить несколько действий.
Как оформлять проверки:
· Нужно создать Excel документ с именем negative_testing_<имя>_<фамилия>.xlsx
· В документе нужно сделать таблицу с четырьмя колонками: ID; Verification; Expected result. Status.
· В колонку «Verification» записывается действие, которое нужно выполнить (нумерованным списком, как в описании шагов для воспроизведения бага).
· В колонку «Expected result» записывается полученный результат (как и в прошлый раз).
· В колонку «Status» нужно вписать passed – если при тестировании полученный результат совпал с ожидаемым. Если полученный результат с ожидаемым не совпал, то надо написать failed, и написать ID бага.

При оценке учитывается:
1. Количество проверок – не менее 9.
2. Корректное оформление.
3. Найдены ли баги. 4. Правильное определение проверки – позитивные проверки заносить не нужно.
5. Оформление на английском языке.
6. Сдача работы вовремя.
